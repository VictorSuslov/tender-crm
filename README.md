# Tender CRM

Автоматизированная система мониторинга и классификации тендерной документации на основе локальной LLM.

## 📋 О проекте

**Tender CRM** — это комплексное решение для строительных организаций, которое автоматизирует обработку входящей электронной почты, классифицирует письма, извлекает тендерные данные из текста и вложений (PDF, DOCX, XLSX), и предоставляет удобный интерфейс для работы с тендерами и AI-поиска по документации.

### Какую проблему решает

Специалисты по тендерам ежедневно тратят **2-3 часа** на:
- Ручной разбор входящей почты
- Фильтрацию спама и рекламных рассылок
- Извлечение ключевых данных из писем и вложений
- Ведение учёта тендеров и связанных писем
- Поиск информации в накопленной документации

**Tender CRM сокращает это время до нескольких минут в день.**

## 🎯 Ключевые возможности

### 📧 Автоматическая обработка почты
- Сбор писем через IMAP (Yandex, Gmail и др.)
- Классификация через локальную LLM: **TENDER / SPAM / GENERAL / EMPTY**
- Извлечение резюме и тендерных данных (НМЦК, сроки, название закупки)
- Парсинг вложений: PDF, DOCX, XLSX, ZIP

### 🤖 AI Поиск (RAG)
- Семантический поиск по документации тендеров
- Генерация ответов на естественном языке
- Указание источников с процентом сходства
- Поиск в контексте конкретного тендера или глобально

###  Десктоп-приложение
- Управление тендерами и письмами
- Связывание писем с тендерами
- AI-ассистент для работы с документацией
- Статистика в реальном времени

### 🔒 Безопасность
- Все данные хранятся локально
- LLM работает на вашем сервере (Ollama)
- Никаких облачных сервисов и утечек данных

## 🚀 Быстрый старт

### Требования

- Windows 10/11 или Linux
- Docker Desktop (для БД)
- Python 3.11+
- Qt 6.10 (для десктоп-клиента)
- Ollama (для LLM)
- ~8 GB свободной оперативной памяти
- GPU с 6+ GB VRAM (рекомендуется для LLM)

### Шаг 1: Развёртывание базы данных

    git clone https://github.com/VictorSuslov/tender-crm-database.git
    cd tender-crm-database
    cp .env.example .env
    # Отредактируйте .env — укажите пароли
    docker compose up -d

Проверка:
    docker compose ps
    # Должны быть запущены tender_crm_db и tender_crm_pgadmin

### Шаг 2: Установка и запуск Backend

    git clone https://github.com/VictorSuslov/tender-crm-backend.git
    cd tender-crm-backend
    python -m venv venv
    venv\Scripts\activate
    pip install -r requirements.txt
    cp .env.example .env
    # Отредактируйте .env — укажите настройки БД, IMAP, Ollama

Установка моделей Ollama:
    ollama pull qwen2.5:7b
    ollama pull jeffh/intfloat-multilingual-e5-large-instruct:q8_0

Запуск:
    python -m app.main

Проверка: http://localhost:8000/docs (Swagger UI)

### Шаг 3: Сборка и запуск Desktop

    git clone https://github.com/VictorSuslov/tender-crm-desktop.git
    # Откройте CMakeLists.txt в Qt Creator
    # Build → Run (Ctrl+R)

Или через CMake:
    mkdir build && cd build
    cmake .. -G "MinGW Makefiles"
    cmake --build .
    ./TenderCrmDesktop.exe

## 🎬 Демо-сценарий (7-10 минут)

### 1. Обработка почты (2 минуты)
- Открыть вкладку **Настройки** → **Обработать почту сейчас**
- Показать, как система обрабатывает реальные письма
- Показать статистику: SPAM / GENERAL / TENDER / EMPTY

### 2. Работа с письмами (2 минуты)
- Открыть вкладку **Письма** → фильтр **🏆 TENDER**
- Выбрать письмо с PDF-вложением
- Показать извлечённую LLM НМЦК (например, 64.8 млн руб.)
- Связать письмо с тендером через диалог

### 3. AI Поиск (3 минуты) — **главный акцент**
- Открыть вкладку **🤖 AI Поиск**
- Выбрать тендер
- Задать вопрос: "Какая НМЦК контракта?"
- Показать ответ с источниками и процентом сходства
- Задать вопрос: "Какие сроки выполнения работ?"
- Показать историю запросов

### 4. Статистика (1 минута)
- Открыть вкладку **Настройки**
- Показать статистику: письма, тендеры, документы, чанки
- Проверить подключение к серверу

## 🎓 Для дипломной работы

### Актуальность
Строительные организации ежедневно получают десятки писем от электронных торговых площадок и заказчиков. Значительная часть — спам и служебные уведомления. Тендерные письма часто содержат важную информацию во вложениях, которую нужно быстро извлечь. Ручная обработка занимает у специалиста до 2-3 часов в день.

### Цель проекта
Разработка системы, которая автоматически обрабатывает входящую почту, классифицирует письма, извлекает тендерные данные из текста и вложений, и предоставляет удобный интерфейс для работы с тендерами и AI-поиска по документации.

### Задачи
1. Реализовать автоматический сбор писем через IMAP
2. Разработать систему классификации писем на основе локальной LLM
3. Реализовать парсинг вложений (PDF, DOCX, XLSX, ZIP)
4. Разработать RAG-систему для семантического поиска по документации
5. Создать десктоп-клиент для удобной работы с системой
6. Обеспечить безопасность данных (локальное хранение и обработка)

### Научная новизна
- Применение локальной LLM для классификации и извлечения данных из тендерной документации
- Использование RAG-подхода для семантического поиска по разнородным документам
- Интеграция векторного поиска (pgvector) с классической реляционной БД

## 🔐 Безопасность
